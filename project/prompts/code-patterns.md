# Code Patterns & Practices
- Keep one feature component or route module per file when practical.
- Prefer colocated styles for feature-specific UI.
- Use TypeScript interfaces or types for shared data contracts.
- Reuse shared utilities instead of duplicating logic.
- Keep route modules readable and small.
