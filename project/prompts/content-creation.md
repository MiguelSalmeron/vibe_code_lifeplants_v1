# Content Creation
- Write copy for the active LifePlants app, not for the legacy static snapshot unless explicitly requested.
- Match the tone of an environmental, community-driven project.
- Keep content clear, warm, and concrete.
- Prefer short headings, direct calls to action, and measurable claims only when the source supports them.
- When bilingual content is needed, keep sections aligned sentence by sentence or block by block.
- When creating different site sections, create routes for them in the \`app/routes\` directory.
- Change \`app/routes.ts\` AFTER all other changes.
- Use kebab-case for file and folder names.
- Google Fonts are specified in the \`links\` export in \`app/root.tsx\`. Avoid using \`@import\` in CSS files to import new ones.
